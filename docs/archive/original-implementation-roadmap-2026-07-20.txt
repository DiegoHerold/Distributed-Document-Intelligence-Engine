# Roadmap de implementação do Eixo

## Regra obrigatória para todas as fases

Toda funcionalidade pública criada em qualquer fase deve seguir o fluxo:

```text
Contrato
   ↓
Implementação no núcleo
   ↓
Caso de uso na camada de aplicação
   ↓
Biblioteca Python
   ↓
API REST
   ↓
CLI, quando útil
   ↓
Testes de paridade
   ↓
Documentação
```

Uma funcionalidade não está concluída apenas porque funciona internamente.

Ela deve estar reutilizável por:

```text
DocumentEngine.local()
DocumentEngine.remote()
API REST
CLI, quando aplicável
```

A API, a biblioteca, os workers e a CLI nunca devem reimplementar a lógica da funcionalidade.
